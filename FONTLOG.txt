FONTLOG for the Pixelspace font family
======================================

This file provides detailed information on the Pixelspace font family.
This information should be distributed along with the Pixelspace fonts
and any derivative works.

Basic Font Information
----------------------

Pixelspace is an original pixel display typeface drawn on a 5x7 cell
grid (with a built-in 1-pixel right-side bearing, effectively 6x8).
It covers the full GF-latin-core glyph set — uppercase A-Z, lowercase
a-z, digits 0-9, common ASCII punctuation, Latin-1 Supplement, Latin
Extended-A/B, general punctuation, currency, combining diacritics, and
a handful of extended Latin letters — plus most of Unicode Box Drawing
(U+2500–257F) and Block Elements (U+2580–259F) for terminal/TUI use.
474 glyphs in total as of v1.100.

The source glyphs live in `sources/glyphs.txt` as a plain-text bitmap
format: each `@glyph NAME` section holds a 7-row × 5-col bitmap (`.`
for off, `#` for on). `tools/build_font.py` parses the bitmap, draws
each "on" pixel as a 110×110-unit rounded square (≈10% corner radius)
with a 15-unit gap to its neighbors, and emits real .ttf, .otf, and
.woff2 files via fontTools. `sources/Pixelspace.svg` is preserved as a
versioned artifact regenerated by `tools/bitmap_to_svg.py`.

Metrics
  UPEM:                 875
  Pixel cell:           125 units
  Pixel body (drawn):   110 units (15-unit gap on the right & below)
  Pixel corner radius:  ~12 units (~10% of the pixel body)
  Advance width:        750 units (5 pixel columns + 1 bearing)
  Ascent:               625 (ink top — 5 rows above baseline)
  Descent:              250 (ink bottom — 2 rows below baseline)
  hhea/sTypo ascent:    1000 (padded for Google Fonts line-metrics rule)
  hhea/sTypo descent:   250
  Line gap:             0    (restored via CSS on specimen)
  Cap height:           625
  x-height:             500
  Underline position:   -188
  Underline thickness:  125

ChangeLog
---------

1 May 2026 (Kumar Anirudha) Pixelspace v1.100
- Terminal coverage: 156 new glyphs covering Unicode Box Drawing
  (U+2500–257F) and Block Elements (U+2580–259F). Drawn in the same
  gappy rounded-pixel style as the Latin glyphs — TUI borders read as
  stitched/dotted lines, btop bar charts as LEGO towers, powerline
  triangles (added in v1.110) as pixel staircases. Brand identity is
  preserved at the cost of conventional terminal-line continuity; see
  build/mockups/comparison.png for the design rationale.
- Source pivot: introduced `sources/glyphs.txt` as the canonical bitmap
  source (plain-text 7×5 grids, one per glyph). The old SVG-font
  `sources/Pixelspace.svg` is now a generated artifact, refreshed via
  `tools/bitmap_to_svg.py`. The migration is round-trip clean — see
  `tools/svg_to_bitmap.py` for the one-shot exporter.
- Build pipeline: set OS/2 ulUnicodeRange2 bits 47 (Box Drawing) and
  48 (Block Elements). Fixed a long-standing head/name version
  mismatch (head.fontRevision was being left at the default 1.0).
- Skipped: the four heavy-double-dash variants (U+254C–254F), and the
  rarest mixed-weight junctions where a 5×7 grid can't usefully
  distinguish them. These can be added in a follow-up if anyone
  notices their absence in real-world TUIs.

27 April 2026 (Kumar Anirudha) Pixelspace v1.001
- Glyph polish: redrew lowercase c, the backtick (`), Ø/ø, and the
  caron/cedilla/dot-above/macron diacritics on c, e, i, t and friends
  (ç, č, ċ, ć, è, é, ê, ë, ē, ė, ě, ī, ť, ț, ȷ) for cleaner pixel
  alignment. Glyph count and metrics unchanged.

24 April 2026 (Kumar Anirudha) Pixelspace v1.000
- Initial release.
- Regular weight only, 318 glyphs, monospace 5x7 pixel design.
- Full GF-latin-core coverage.
- Compiled from sources/Pixelspace.svg via fontTools.
- Rounded-square pixels (10% radius) with 15-unit inter-pixel gap.
